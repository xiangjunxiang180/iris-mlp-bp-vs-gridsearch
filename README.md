# iris-mlp-bp-vs-gridsearch

## 项目描述
本项目实现了两种不同训练方式的三层多层感知器（MLP），用于对经典的Iris鸢尾花数据集进行分类任务，并系统性对比两种方式的训练逻辑、效率及分类效果：
- 实验一：纯手写MLP（基于NumPy），不依赖任何深度学习框架，通过**随机采样模拟网格搜索**的方式寻找最优权重（无反向传播）；
- 实验二：基于PyTorch搭建MLP，利用框架的自动微分能力实现**反向传播（BP）**，并通过随机梯度下降（SGD）优化器迭代更新参数。

通过对比两个实验的结果，揭示反向传播在神经网络训练中的核心价值，以及“纯手写底层实现”与“框架化工程实现”的差异。

## 实验背景
### 数据集说明
Iris数据集包含150条样本，每条样本有4个数值型特征（花萼长度、花萼宽度、花瓣长度、花瓣宽度），分为3个类别（Setosa、Versicolor、Virginica），是多分类任务的经典测试集。

### 统一MLP网络结构
两个实验均采用完全一致的三层MLP结构：
- 输入层：4个神经元（对应4个特征）；
- 隐藏层：5个神经元，使用Sigmoid激活函数引入非线性；
- 输出层：3个神经元（对应3个类别）。

## 实验一：纯手写MLP + 网格搜索（非BP）
### 实现要点
- 无框架依赖：完全基于NumPy手动实现Sigmoid激活函数、MLP前向传播（矩阵乘法+偏置+激活函数）；
- 非BP训练逻辑：不计算梯度、不迭代更新参数，通过“随机采样权重组合”模拟网格搜索，仅保留训练集准确率最高的权重；
- 权重搜索范围：权重/偏置取值范围为 `[-0.8, 0.8]`，通过固定次数（50次）随机采样覆盖极小部分权重空间；
- 功能模块化：数据加载、激活函数、前向传播、权重搜索、测试、可视化等功能解耦为独立函数。

### 核心流程
1. 加载Iris数据集，标准化后按7:3划分训练集/测试集；
2. 循环随机生成权重组合（w1/b1：输入→隐藏层；w2/b2：隐藏→输出层）；
3. 前向传播计算训练集预测结果，保留训练准确率最高的权重；
4. 用最优权重测试测试集，可视化“试错过程”和分类结果。

## 实验二：PyTorch实现BP神经网络
### 实现要点
- 框架赋能：基于PyTorch搭建MLP，利用`Autograd`自动实现反向传播的梯度计算；
- 标准BP训练流程：前向传播计算损失 → 反向传播计算梯度 → 优化器（SGD）更新参数，迭代训练；
- 训练监控：记录每轮训练的损失值和准确率，支持训练过程可视化；
- 评估优化：测试时禁用梯度计算（`torch.no_grad()`），提升推理效率。

### 核心流程
1. 加载并标准化数据，转换为PyTorch张量；
2. 定义MLP网络结构（`nn.Sequential`堆叠层）；
3. 定义交叉熵损失函数（适配多分类）和SGD优化器；
4. 迭代训练：前向传播→计算损失→梯度清零→反向传播→参数更新；
5. 测试集评估，可视化训练损失/准确率曲线、测试集分类结果。

## 实验结果核心差异
| 对比维度            | 实验一（纯手写 + 网格搜索）| 实验二（PyTorch + BP）|
|---------------------|---------------------------------------------|---------------------------------------------|
| 训练原理            | 暴力试错（随机采样权重），无梯度优化        | 梯度定向优化（BP），沿损失下降方向更新参数  |
| 训练效率            | 极低：50次尝试仅遍历极小部分权重空间        | 极高：100轮迭代即可快速收敛                 |
| 训练准确率稳定性    | 波动大：最佳训练准确率70%-90%（非稳定）| 稳定：100轮后训练准确率接近100%             |
| 测试准确率          | 较低且不稳定（60%-85%），无优化空间         | 高且稳定（95%-100%），泛化能力强            |
| 代码复杂度          | 高：手动实现矩阵运算、前向传播等所有逻辑    | 低：框架封装底层，聚焦训练流程              |
| 可扩展性            | 差：权重维度爆炸，无法适配复杂网络/大数据   | 强：可扩展到深层网络、批量训练、高级优化器  |
| 训练过程可解释性    | 简单：仅记录“试错”后的最佳准确率            | 丰富：可监控损失/准确率变化，分析收敛趋势   |

### 关键差异补充说明
1. **训练逻辑本质**：
   实验一的“网格搜索”是暴力枚举，Iris任务的权重空间维度为 `4×5 + 5 + 5×3 + 3 = 43`，50次尝试仅能覆盖极小部分，无法找到全局最优；
   实验二的BP是“定向优化”，通过梯度指引参数迭代更新，能快速收敛到最优解。

2. **准确率表现**：
   实验一的测试准确率受随机权重限制，难以突破90%；
   实验二的BP训练能充分拟合数据，测试准确率稳定在95%以上（甚至100%）。

3. **工程实现成本**：
   实验一纯手写代码需处理矩阵维度匹配、数值稳定性等细节，易出错；
   实验二利用PyTorch的封装能力，代码更简洁、鲁棒性更强。
